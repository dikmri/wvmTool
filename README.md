# wvmTool

ブラウザ完結型の動画モザイク編集Webアプリです。動画ファイルはサーバーにアップロードされず、すべてローカルブラウザ内で処理されます。

## GitHub Pages URL

https://dikmri.github.io/wvmTool/

## 対応ブラウザ

- Google Chrome 最新版 (推奨)
- Microsoft Edge 最新版
- Firefox 最新版
- Safari: 一部機能が制限される場合があります（WebCodecs/WebGL2の互換性）

## 主な機能

- MP4動画のドラッグ&ドロップ読み込み
- 複数の矩形モザイクトラックを独立して管理
- モザイク範囲の描画・移動・コーナーハンドルによるリサイズ
- モザイク範囲の回転（Q/E キー、R でリセット）
- キーフレームベースのモザイク位置・サイズ・回転アニメーション（キーフレーム間は線形補間）
- WebGL2によるリアルタイムモザイクプレビュー（回転対応）
- WebCodecsによる高速フレームデコード/エンコード
- モザイク適用済み動画のMP4書き出し（ファイル名に `_mosaic` を付与）

## 使い方

1. MP4動画をビューポートにドラッグ&ドロップする（またはクリックしてファイル選択）
2. ツールパネルで「＋ 追加」をクリックしてモザイクトラックを作成する
   - 追加すると自動的に描画モードへ切り替わる
3. ビューポート上でドラッグしてモザイク範囲を描く
4. タイムラインをシークしながら、必要な位置でキーフレームを追加する（K キー）
5. 必要に応じて選択モードに切り替え、矩形をドラッグして移動・コーナーハンドルでリサイズ
6. 書き出しパネルで「書き出し開始」をクリックする

### モザイクトラックの操作

- **複数トラック**: トラックを複数追加すると、それぞれ独立したモザイク領域を持つ
- **●/○ ボタン**: トラックの有効/無効を切り替え
- **× 削除**: 選択中のトラックを削除
- **モザイクサイズ**: スライダーでモザイクの粒度を調整（5〜80px）

### キーボードショートカット

| キー | 操作 |
|------|------|
| Space | 再生/停止 |
| ArrowLeft | 1フレーム戻る |
| ArrowRight | 1フレーム進む |
| K | 現在位置にキーフレーム追加 |
| Delete | 選択中キーフレーム削除 |
| Q | モザイク範囲を反時計回りに5°回転 |
| E | モザイク範囲を時計回りに5°回転 |
| R | モザイク範囲の回転をリセット（0°） |

### 書き出し設定

| 設定 | 内容 |
|------|------|
| コーデック | Auto (H.264優先) / H.264 / VP9 / AV1 |
| 接尾辞 | 出力ファイル名に付加する文字列（デフォルト: `_mosaic`） |

## プライバシー・セキュリティ

**動画ファイルはサーバーにアップロードされません。** すべての処理はユーザーのブラウザ内で完結します。外部APIへの通信は一切行いません。

## 既知の制限

- 音声は書き出しに含まれません（映像のみ）
- Safari では WebCodecs / WebGL2 の対応状況により一部機能が制限されます
- 非常に長尺・高解像度の動画ではメモリ不足になる場合があります
- 書き出しにはブラウザの WebCodecs API が必要です

## 技術構成

| レイヤー | 技術 |
|----------|------|
| UIフレームワーク | Svelte 5 + TypeScript |
| ビルドツール | Vite 6 |
| 動画デコード/エンコード | WebCodecs API |
| レンダリング | WebGL2（回転対応モザイクシェーダー） |
| Canvas2Dフォールバック | WebGL2非対応環境向け（回転対応） |
| バックグラウンド処理 | Web Worker + OffscreenCanvas |
| MP4コンテナ解析 | mp4box.js |
| MP4マルチプレクス | mp4-muxer |
| ホスティング | GitHub Pages |

## パフォーマンス方針

- UIスレッドに動画フレームデータを持たせない
- 重い処理（デコード・エンコード・モザイク適用）はすべてWeb Workerで実行
- WebGL2シェーダーによるGPUモザイク処理（最大8トラック同時）
- OffscreenCanvasを使用したメインスレッドへの影響最小化
- VideoFrame は使用後に必ず `close()` してメモリリークを防止
- 書き出し時は全フレームを保持せず逐次処理

## 開発方法

```bash
# 依存関係のインストール
npm install

# 開発サーバー起動
npm run dev

# プロダクションビルド
npm run build

# ビルドのプレビュー
npm run preview
```

## デプロイ方法

`main` ブランチへの push で GitHub Actions が自動的にビルドし GitHub Pages へデプロイします。

手動デプロイする場合:

```bash
npm run build
# dist/ ディレクトリの内容を GitHub Pages にデプロイ
```
