# wvmTool

ブラウザ完結型の動画モザイク編集Webアプリです。動画ファイルはサーバーにアップロードされず、すべてローカルブラウザ内で処理されます。

## GitHub Pages URL

https://dikmri.github.io/wvmTool/

## 対応ブラウザ

- Google Chrome 最新版 (推奨)
- Microsoft Edge 最新版
- Firefox 最新版
- Safari: 一部機能が制限される場合があります（WebCodecs/WebGL2の互換性）

## 主な機能

- MP4動画のドラッグ&ドロップ読み込み
- 矩形モザイク範囲の追加・移動・リサイズ
- キーフレームベースのモザイク位置アニメーション
- WebGL2によるリアルタイムモザイクプレビュー
- WebCodecsによる高速フレームデコード/エンコード
- モザイク適用済み動画のMP4書き出し（ファイル名に `_mosaic` を付与）

## 使い方

1. MP4動画をビューポートにドラッグ&ドロップする
2. ツールパネルで「＋ 追加」をクリックしてモザイクトラックを作成する
3. 描画モード（□ 描画）に切り替え、ビューポート上でモザイク範囲をドラッグして描く
4. タイムラインをシークしながら、必要な位置でキーフレームを追加する（K キー）
5. 書き出しパネルで「書き出し開始」をクリックする

### キーボードショートカット

| キー | 操作 |
|------|------|
| Space | 再生/停止 |
| ArrowLeft | 1フレーム戻る |
| ArrowRight | 1フレーム進む |
| Shift + ArrowLeft | 10フレーム戻る |
| Shift + ArrowRight | 10フレーム進む |
| K | 現在位置にキーフレーム追加 |
| Delete | 選択中キーフレーム削除 |

## プライバシー・セキュリティ

**動画ファイルはサーバーにアップロードされません。** すべての処理はユーザーのブラウザ内で完結します。外部APIへの通信は一切行いません。

## 既知の制限

- 書き出し時の音声保持は実験的機能です（現バージョンでは映像のみ書き出し）
- Safari では WebCodecs / WebGL2 の対応状況により一部機能が制限されます
- 非常に長尺・高解像度の動画ではメモリ不足になる場合があります
- 書き出しにはブラウザの WebCodecs API が必要です

## 技術構成

| レイヤー | 技術 |
|----------|------|
| UIフレームワーク | Svelte 5 + TypeScript |
| ビルドツール | Vite 6 |
| 動画デコード/エンコード | WebCodecs API |
| レンダリング | WebGL2 |
| バックグラウンド処理 | Web Worker + OffscreenCanvas |
| MP4コンテナ解析 | mp4box.js |
| MP4マルチプレクス | mp4-muxer |
| ホスティング | GitHub Pages |

## パフォーマンス方針

- UIスレッドに動画フレームデータを持たせない
- 重い処理（デコード・エンコード・モザイク適用）はすべてWeb Workerで実行
- WebGL2シェーダーによるGPUモザイク処理
- OffscreenCanvasを使用したメインスレッドへの影響最小化
- VideoFrame は使用後に必ず `close()` してメモリリークを防止
- 書き出し時は全フレームを保持せず逐次処理

## 開発方法

```bash
# 依存関係のインストール
npm install

# 開発サーバー起動
npm run dev

# プロダクションビルド
npm run build

# ビルドのプレビュー
npm run preview
```

## デプロイ方法

`main` ブランチへの push で GitHub Actions が自動的にビルドし GitHub Pages へデプロイします。

手動デプロイする場合:

```bash
npm run build
# dist/ ディレクトリの内容を GitHub Pages にデプロイ
```
