这是一个包含浏览器的视频马赛克编辑网络应用程序。

## 可从以下 URL 获取

https://dikmri.github.io/wvmTool/

## 兼容的浏览器

- Google Chrome 最新版本（推荐）
- 微软Edge最新版本
- 最新版本的火狐浏览器
- Safari：某些功能可能受到限制（WebCodecs/WebGL2 兼容性）

## 主要特点

- 拖放加载 MP4 视频
- 独立管理多个矩形马赛克轨道
- 使用角手柄绘制、移动马赛克区域并调整其大小
- 旋转马赛克范围（Q/E键，用R重置）
- 基于关键帧的马赛克位置、大小、旋转、显示/隐藏动画
- 使用WebGL2实时马赛克预览（支持旋转）
- 使用 WebCodecs 进行快速帧解码/编码
- 导出带音频的 MP4（自动继承原始视频的比特率）
- 播放时可以隐藏叠加边框并查看纯马赛克
- 通过屏幕上的“如何使用”按钮显示快捷方式列表
- **多语言支持**：日语/英语/中文/한국어/Español/Français（带标题切换）

## 如何使用

1. 将 MP4 视频拖放到视口（或单击选择文件）
2.点击工具面板中的“+添加”创建马赛克轨道
- 添加时自动切换到绘图模式
3.通过在视口上拖动来绘制马赛克区域
4. 在时间轴上搜索的同时在所需位置添加关键帧（K 键）
5. 如有必要，切换到选择模式，拖动矩形进行移动，然后使用角手柄调整其大小。
6.在导出面板中点击“开始导出”

### 马赛克轨道操作

- **多个轨道**：如果添加多个轨道，每个轨道将有自己独立的马赛克区域。
- **●/○ 按钮**：启用/禁用轨道
- **删除**：删除所选曲目
- **马赛克大小**：使用滑块调整马赛克的粒度（5 至 80 像素）

### 键盘快捷键

|关键|运营|
|------|------|
|空间|播放/停止 |
|向左箭头 |返回一帧 |
|向右箭头|前进 1 帧 |
| Shift+← / → |移至第一帧/最后一帧 |
|克 |在当前位置添加关键帧 |
|删除 |删除选定的关键帧 |

|电子|顺时针旋转马赛克范围5° |

> 播放时，马赛克范围的边界被隐藏，只能看到马赛克效果。

### 导出设置

|设置|内容 |
|------|------|
|编解码器|自动（H.264 优先）/H.264/VP9/AV1 |
|图像质量|最高图像质量（量化器 16）/高图像质量（22）/标准（28）/低图像质量（35）。默认为“高质量” |
|后缀 |要附加到输出文件名的字符串（默认值：`_mosaic`）|

在 H.264 中，使用 VBR（可变比特率）控制质量。自动检测原始视频的比特率，并将其乘以质量设置乘数以确定目标比特率（高质量=与原始视频相同，最高质量=1.5x，标准=0.65x，低质量=0.35x）。 FPS 是从原始视频中自动检测的。音频是从原始视频传递过来的。

如果添加马赛克轨​​道并在绘图模式下绘制矩形，则绘制完成后会自动切换到选择模式。也可以在视频范围之外指定马赛克范围。

## 隐私/安全

**视频文件未上传到服务器。 ** 所有处理均在用户浏览器内完成。没有与外部 API 的通信。

## 已知限制

- 根据 WebCodecs/WebGL2 兼容性，某些功能在 Safari 中可能会受到限制。
- 很长且高分辨率的视频可能会耗尽内存。
- 导出需要浏览器的WebCodecs API

## 技术配置

|层 |技术 |
|----------|------|
|用户界面框架 | Svelte 5 + TypeScript |
|构建工具|投票 6 |
|视频解码/编码 | WebCodecs API |
|渲染| WebGL2（旋转兼容马赛克着色器）|
| Canvas2D 后备 |对于不支持WebGL2的环境（支持旋转）|
|后台处理 |网络工作者 + OffscreenCanvas |
| MP4容器分析| mp4box.js |
| MP4多路复用| mp4 复用器 |
|托管| GitHub 页面 |

## 绩效政策

- UI线程中没有视频帧数据
- 所有繁重的处理（解码、编码、马赛克应用）均由 Web Worker 执行
- 使用 WebGL2 着色器进行 GPU 马赛克处理（最多同时处理 8 个轨道）
- 使用 OffscreenCanvas 最大限度地减少对主线程的影响
- 使用后始终“close()”VideoFrame，以防止内存泄漏
- 导出时，所有帧不会保留并按顺序处理
